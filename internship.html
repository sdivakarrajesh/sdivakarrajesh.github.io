MY First Internship!

What I did there as an intern?


I worked on integrating SAML authentication to a webApp. If you have looked carefully when you try to login using Google or any other services you might have come across OAuth2
SAML is very similar to that.

What is SAML Authentication?


SAML - Security Assertion Markup Language
It is used to provide authentication and authorization between services. Typically a Service provider and an Identity provider It is an XML - based markup language for security assertions. It also facilitates Single Sign On(SSO) where the user's authentication information persists across different browser sessions. Who is a service provider? It is the service or product that you are trying to access. Example: Trying to access an online video-streaming service Who is an Identity provider? An identity provider is the one who authenticates that the user is who he/she claims to be, by verifying the creditials, commonly a username and a password.

What are the specifics?

> I was asked to implement a test webApp and deploy it in a TomCat Server.
> The user has to sign-in using any one his favorite Identity from the options available . > This causes the browser has to create a SAML request and redirect to the identity providers site to carry out the authentication process. > After authentication based on the SAML reponse, the user's role has to extracted from the response and has to be redirected based on the role(Admin or User) and land on different pages.

OneLogin as an Identity provider


OneLogin has a sample tutorial written for java here that explains all the necessary steps that are to followed to get a sample application up and running
OneLogin also has a toolkit that handles all the hard-lifting such as creating a SAML request, parsing and validating a SAML reponse and all that. In no time I was able to set up a sample application that has oneLogin sign-in One great issue that had while implementing oneLogin is that if you opt fingerPrint aside the X.509 certificate, the fingerprint has to be unformatted or else you may get an idp/ certificate error You can use this simple tool to get your unformatted fingerprint for the fingerPrint that you get in the admin console

Okta as an Identity provider


Unlike OneLogin I wasn't able to find an out-of-the-box toolkit for Okta. So, I decided to create the SAML request and parse and validate the reponse on my own. Then After a lot of surfing i found that there are plenty of libraries out there that eases out the task and opted for openSAML With that I was able to get everything right very quickly.
